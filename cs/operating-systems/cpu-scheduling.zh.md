---
title: CPU 调度
---

## 1. What
CPU 调度决定“当前就绪的任务里，下一步由谁占用处理器执行”。这是操作系统最核心的控制问题之一，因为 CPU 时间是被所有任务共享的稀缺资源。

经典调度算法包括 FCFS、SJF、SRTF、优先级调度、时间片轮转以及多级反馈队列。

## 2. Why
调度之所以重要，是因为不同策略优化的目标并不相同：

- 更低的平均等待时间
- 更快的交互响应
- 更好的公平性
- 更高的 CPU 利用率
- 在混合工作负载下保持稳定服务

调度器不只是回答“谁先跑”，而是在吞吐、延迟和公平之间强制执行系统级取舍。

## 3. How
常见评价指标包括：

- 周转时间
- 等待时间
- 响应时间
- 吞吐量
- CPU 利用率

代表性算法如下：

| Algorithm | Idea | Strength | Risk |
| --- | --- | --- | --- |
| FCFS | 按到达顺序执行 | 简单 | convoy effect |
| SJF | 选最短作业 | 平均等待时间低 | 需要预估执行时间 |
| SRTF | 可抢占的 SJF | 响应更好 | 切换更多 |
| Priority | 先跑高优先级 | 策略灵活 | 可能饥饿 |
| Round Robin | 按时间片轮转 | 适合交互 | 时间片难调 |

时间片轮转的核心流程：

```text
put all ready tasks in queue
run head task for one quantum
if task finishes:
  remove it
else:
  move it to queue tail
```

多级反馈队列则会根据任务表现动态调整优先级，通常让短作业和交互任务更快得到响应，同时保证长时间 CPU 密集型任务也能继续推进。

## 4. Better
没有一种调度算法适合所有负载。FCFS 非常简单，但当长作业排在前面时，会拖慢后续大量短作业。SJF 在理论上能最小化平均等待时间，但现实系统通常无法准确提前知道作业时长。

时间片轮转更适合分时系统，因为它能让每个进程周期性获得 CPU。优先级调度便于实现差异化服务，但如果没有 aging 机制，低优先级任务可能长期得不到执行。

现代操作系统通常采用混合调度器，因为真实负载往往同时包含交互任务、后台计算和内核活动。

## 5. Beyond
生产级调度还必须处理：

- 多核负载均衡
- 缓存亲和性
- 实时截止时间
- NUMA 局部性
- 虚拟化层带来的额外调度

更广泛的结论是：调度是“用机制实现策略”。算法该如何选，最终取决于系统向用户承诺的是公平、响应、可预测性，还是纯吞吐。
