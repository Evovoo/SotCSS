---
title: 内存管理
---

## 1. What
内存管理是操作系统用来完成内存分配、保护、地址转换和回收的一整套机制。它把程序看到的逻辑地址空间，与硬件上的物理内存和外部存储层级连接起来。

教材中的核心主题包括分页、分段、虚拟内存、页面置换以及 LRU 这类基于局部性的策略。

## 2. Why
这个主题之所以关键，是因为内存既有限又共享：

- 多个进程必须安全共存
- 程序需要的地址空间往往大于物理 RAM
- 重定位和保护必须可靠实现
- 性能高度依赖局部性和缺页行为

内存管理正是正确性、隔离性与性能三者交汇的地方。

## 3. How
分页把逻辑内存划分为固定大小的页，把物理内存划分为同样大小的页框。地址转换过程通常是：

```text
logical address = page number + offset
page number -> page table -> frame number
physical address = frame number + offset
```

分段则按逻辑意义划分内存，例如代码段、数据段、栈段。段是可变长的，因此更贴近程序员理解的程序结构。

虚拟内存允许进程只有一部分页面驻留在 RAM 中，不活跃页面则留在辅存中。发生缺页时的大致过程是：

```text
access page
if page not in memory:
  trap to OS
  load page from disk
  possibly evict another page
  resume execution
```

LRU 的思想是淘汰“最长时间未被访问”的页面，用来近似利用程序的局部性规律。

## 4. Better
分页用固定大小单元管理内存，避免了外部碎片，也更利于统一分配；但它会引入内部碎片。分段更贴合程序结构，但可变长分配更容易产生碎片，管理也更复杂。

虚拟内存优于“所有页面必须常驻内存”的方式，因为它提高了利用率并支持更强隔离；但如果缺页过于频繁，就会进入抖动。LRU 比 FIFO 更符合局部性直觉，但精确实现 LRU 成本较高，因此实际系统常用 Clock 等近似算法。

最关键的区别在于：程序看到的是一个抽象的大地址空间，而硬件真正提供的是有限的物理 RAM。

## 5. Beyond
现代内存管理还会涉及：

- TLB 行为
- 大页
- 写时复制
- NUMA 感知分配
- 内存映射文件

更深层的结论是：内存不只是“可存放数据的地方”，它本质上还是一个延迟层级。好的内存管理，依赖于把保护、转换和局部性规律结合起来。
