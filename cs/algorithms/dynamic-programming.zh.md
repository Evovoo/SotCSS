---
title: 动态规划
---

## 1. What
动态规划用于解决具有重叠子问题和最优子结构的问题。它的核心思想是把中间结果保存下来，避免重复计算，从而把原本指数级的重复搜索压缩成系统化的多项式求解。

经典问题包括背包问题、最长公共子序列、最长递增子序列、编辑距离以及很多序列优化问题。

## 2. Why
动态规划之所以重要，是因为很多朴素递归会反复求解相同状态：

- 指数级搜索常能被大幅降低
- 状态转移能把问题结构明确表达出来
- 它既能做最优化，也能做计数和可行性判断
- 很多序列与资源分配问题天然符合 DP 形态

关键不在于“开一个表”，而在于定义对状态和转移。

## 3. How
动态规划的一般步骤是：

1. 定义状态
2. 推出状态转移
3. 明确边界条件
4. 选择自顶向下记忆化还是自底向上填表

0/1 背包：

```text
dp[i][w] = using first i items with capacity w, the max value
```

转移方程：

```text
if weight[i] > w:
  dp[i][w] = dp[i - 1][w]
else:
  dp[i][w] = max(dp[i - 1][w], dp[i - 1][w - weight[i]] + value[i])
```

最长公共子序列：

```text
if s1[i] == s2[j]:
  dp[i][j] = dp[i - 1][j - 1] + 1
else:
  dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])
```

常见 DP 形态：

| Style | Idea |
| --- | --- |
| memoization | 自顶向下递归 + 缓存 |
| tabulation | 自底向上迭代填表 |
| rolling array | 利用依赖局部性压缩空间 |

## 4. Better
当子问题存在大量重叠时，DP 明显优于朴素递归。它和贪心的区别在于：贪心立刻做局部最优决策，而 DP 会显式比较多个可能状态后再决定。

和回溯相比，DP 在“许多搜索分支最终落到同一状态”时更强；和分治相比，DP 关注的是复用，而不是子问题独立。

但如果状态设计得太大，即使转移正确，DP 也可能完全不可用。很多时候真正的难点不是写代码，而是避免状态空间爆炸。

## 5. Beyond
更深入的 DP 技术包括：

- 状态压缩
- 单调队列优化或斜率优化
- 区间 DP
- 树形 DP
- 位压 DP

更深层的结论是：动态规划首先是一种建模方法，而不是固定套路。大部分难度都在于识别正确的状态表示，而不是最后那几层循环。
