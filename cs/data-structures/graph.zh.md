title: 图
---

## 1. What
图用来表示顶点之间的两两关系。和树不同，图允许任意连接、环以及多条路径，因此非常适合描述网络、依赖关系、地图和社交连接。

最基础的存储方式是邻接矩阵与邻接表，最基础的遍历方式是 DFS 和 BFS。

## 2. Why
当问题核心是“连接关系”而不是“线性顺序”或“层次结构”时，图就变得关键：

- 路径规划
- 依赖分析
- 推荐系统与社交网络
- 编译器控制流图或调用图

图的价值在于它足够通用，能自然承载可达性、最短路、连通分量和生成树等问题。

## 3. How
邻接矩阵的特点：

- 使用 `V x V` 的二维表
- 判断一条边是否存在是 `O(1)`
- 空间开销是 `O(V^2)`
- 更适合稠密图

邻接表的特点：

- 为每个顶点保存邻居列表
- 空间开销是 `O(V + E)`
- 遍历邻居更高效
- 更适合稀疏图

DFS 更强调“走深”，通常写成递归或显式栈：

```text
dfs(v):
  mark v visited
  for each neighbor u of v:
    if u not visited:
      dfs(u)
```

BFS 更强调“按层扩展”，依赖队列：

```text
bfs(start):
  mark start visited
  enqueue(start)
  while queue not empty:
    v = dequeue()
    for each neighbor u of v:
      if u not visited:
        mark u visited
        enqueue(u)
```

## 4. Better
邻接矩阵更适合下面这些场景：

- 图很稠密
- 需要频繁做边存在性判断
- 顶点数量不大且相对固定

邻接表则更适合：

- 图比较稀疏
- 主要工作是遍历
- 更在意内存效率

DFS 适合做拓扑推理、环检测和结构探索；BFS 则更适合无权图最短路径和按层发现问题。

## 5. Beyond
真实的图算法会很快扩展到加权图、最小生成树、最短路径、强连通分量和网络流。

这里最重要的工程结论是“表示决定复杂度表现”：同一个抽象图，如果工作负载偏查边、偏遍历、偏稀疏、偏稠密或需要动态更新，最合适的实现方式会完全不同。
