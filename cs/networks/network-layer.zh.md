---
title: 网络层
---

## 1. What
网络层负责在多条链路和多个中间路由器之间完成逻辑编址与分组转发。在互联网中，这一角色的核心协议就是 IP。

这一层的核心内容包括 IP 分组传送、子网划分、转发、分片相关概念以及决定分组如何穿越网络的路由算法。

## 2. Why
网络层之所以重要，是因为通信通常跨越很多局部网络，而不是停留在单一链路内部：

- 主机需要全局可解释的逻辑地址
- 路由器必须决定下一跳
- 路径选择会影响可达性和效率
- 不同链路技术之间需要统一的转发抽象

没有网络层，通信就只能停留在本地一跳范围内。

## 3. How
IP 提供的是一种尽力而为、无连接的分组服务。它不保证送达、不保证顺序，也不保证时延；它主要定义了分组如何被编址和转发。

IP 的关键点包括：

- 源 IP 与目的 IP 地址
- TTL 或 hop limit，用于防止无限转发
- 路由表，用于做下一跳决策
- 子网掩码或前缀，用于地址聚合

路由器转发流程可以概括为：

```text
receive packet
inspect destination IP
perform longest-prefix match in routing table
choose next hop and outgoing interface
decrement TTL
forward packet
```

路由算法常见两类：

- 距离向量：交换“到各目标的距离估计”
- 链路状态：先获知拓扑，再自行计算最短路径

可以概括为：

| Routing Style | Core Idea |
| --- | --- |
| distance-vector | “每个目标离我多远？” |
| link-state | “网络拓扑长什么样，我该怎么算路径？” |

## 4. Better
和传输层相比，IP 故意保持简单，因为它优先追求大规模互联和协议兼容，而不是端到端可靠保证。这使它能适配很多链路技术和上层协议。

距离向量路由实现更简单，但在收敛速度和 count-to-infinity 等问题上较弱；链路状态路由掌握的信息更完整，通常收敛更稳定，但代价是需要更大范围分发拓扑信息并做更多计算。

这里最重要的区分是：转发和路由不是一回事。转发是本地动作，决定一个包下一步往哪发；路由则是控制过程，用来决定整个网络中的下一跳规则应当是什么。

## 5. Beyond
现代网络层实践还会进一步涉及：

- IPv4 与 IPv6
- NAT 与地址转换
- MPLS 与流量工程
- 基于策略的路由
- SDN 控制器对转发表的影响

更深层的结论是：网络层是在“不完整信息”和“动态拓扑”约束下提供可达性的机制。它的设计本质上是在可扩展性、简洁性和控制能力之间做平衡。
